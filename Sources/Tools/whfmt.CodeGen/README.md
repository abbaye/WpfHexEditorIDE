# whfmt.CodeGen

**Generate strongly-typed C# parser classes** from `.whfmt` binary format definitions.

```bash
dotnet tool install -g whfmt.CodeGen
whfmt-codegen generate PNG --namespace My.Parsers --output PngParser.cs
```

Every `BlockDefinition` in a `.whfmt` file becomes a **typed property** in the generated class.  
`valueMap` entries become `switch` expressions.  
Big-endian fields emit byte-swap helpers automatically.

Powered by **790+ whfmt.FileFormatCatalog definitions**.

---

## Install

```bash
dotnet tool install -g whfmt.CodeGen
```

## Commands

### `whfmt-codegen generate`

```
whfmt-codegen generate <format> [options]
```

| Option | Short | Default | Description |
|---|---|---|---|
| `<format>` | | *required* | Format name, extension, or path to `.whfmt` file |
| `--namespace` | `-n` | `Generated.Parsers` | C# namespace for the generated class |
| `--class` | `-c` | `<FormatName>Parser` | Class name override |
| `--output` | `-o` | stdout | Output file path |
| `--validate` | | false | Emit signature / checksum assertions |
| `--async` | | false | Generate `async Task<T>` overloads |

### `whfmt-codegen list`

```
whfmt-codegen list [--search <term>] [--category <name>]
```

Lists all binary formats available for code generation.

```
  Archives
    ZIP                                      .zip, .jar, .apk, .aar
    7-ZIP                                    .7z
    RAR                                      .rar, .r00

  Audio
    MP3                                      .mp3
    FLAC                                     .flac

  Images
    PNG                                      .png
    JPEG                                     .jpg, .jpeg
```

## Examples

### Generate a PNG parser

```bash
whfmt-codegen generate PNG \
  --namespace My.Imaging \
  --output src/Parsers/PngParser.cs \
  --validate
```

**Output** (`PngParser.cs`):

```csharp
// <auto-generated>
//   Generated by whfmt-codegen from 'PNG' v1.0
//   Category: Images
//   Portable Network Graphics image format
//   Do not edit — regenerate with: whfmt-codegen generate
// </auto-generated>

using System;
using System.IO;
using System.Threading;
using System.Threading.Tasks;

namespace My.Imaging;

/// <summary>Strongly-typed parser for PNG files (generated from .whfmt definition).</summary>
public sealed class PngParser
{
    /// <summary>PNG 8-byte file signature</summary>
    public byte[] Signature { get; private set; }

    /// <summary>Image width in pixels</summary>
    public uint Width { get; private set; }

    /// <summary>Image height in pixels</summary>
    public uint Height { get; private set; }

    /// <summary>Bits per channel</summary>
    public byte BitDepth { get; private set; }

    /// <summary>PNG color type</summary>
    public byte ColorType { get; private set; }

    public string ColorTypeLabel => ColorType switch {
        0 => "Grayscale",
        2 => "RGB",
        3 => "Indexed",
        4 => "Grayscale+Alpha",
        6 => "RGBA",
        _ => "Unknown"
    };

    /// <summary>Parse a PNG from a stream.</summary>
    public static PngParser Parse(Stream stream) { ... }

    /// <summary>Parse a PNG from a byte array.</summary>
    public static PngParser Parse(byte[] data) => Parse(new MemoryStream(data));

    /// <summary>Parse a PNG from a file path.</summary>
    public static PngParser ParseFile(string path) { ... }
}
```

### Generate an async PE parser

```bash
whfmt-codegen generate PE_EXE \
  --namespace My.Native \
  --class PortableExecutableParser \
  --async \
  --validate \
  --output src/Parsers/PortableExecutableParser.cs
```

### Generate from a local `.whfmt` file

```bash
whfmt-codegen generate ./MyCustomFormat.whfmt \
  --namespace Acme.Formats \
  --output src/Parsers/MyCustomFormatParser.cs
```

### List formats in a category

```bash
whfmt-codegen list --category Images
whfmt-codegen list --search "sqlite"
```

## Usage in Code (as a library)

`whfmt.CodeGen` is a dotnet global tool. To generate parsers programmatically at build time, use the `WhfmtCodeGen.Generator.ParserGenerator` class directly by referencing the tool assembly, or invoke it as a CLI step in your MSBuild pipeline:

```xml
<!-- MyProject.csproj -->
<Target Name="GenerateParsers" BeforeTargets="Build">
  <Exec Command="whfmt-codegen generate PNG -n $(RootNamespace).Parsers -o $(ProjectDir)Generated/PngParser.cs" />
</Target>
```

## Generated Class Shape

For each format, `whfmt-codegen` emits:

| .whfmt block | Generated C# |
|---|---|
| `bytes` / `uint8`…`uint64` | Typed property (`byte`, `ushort`, `uint`, `ulong`) |
| `string` / `ascii` / `utf8` | `string` property with null-trim |
| `valueMap` | `switch` expression property returning human label |
| `isSignature: true` | Assertion comment (with `--validate`) |
| `endian: "big"` | BSwap16/32/64 inline helper |
| No field length | `byte[]` fallback |

### Async overloads (`--async`)

```csharp
Task<PngParser> ParseAsync(Stream stream, CancellationToken ct = default)
Task<PngParser> ParseFileAsync(string path, CancellationToken ct = default)
```

## Architecture

```
whfmt.CodeGen (dotnet global tool: whfmt-codegen)
├── Commands/
│   ├── GenerateCommand   — resolve format → call ParserGenerator → write output
│   └── ListCommand       — catalog browser with search/category filters
└── Generator/
    └── ParserGenerator   — JSON → C# source code generator
```

Depends on: `whfmt.FileFormatCatalog 1.3.0+` · `System.CommandLine 2.0.0-beta4` — cross-platform net8.0.
